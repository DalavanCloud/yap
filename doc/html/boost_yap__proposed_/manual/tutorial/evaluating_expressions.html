Evaluating Expressions

Boost.YAP expressions are evaluated explicitly and implicitly:

Boost.YAP expressions are explicitly evaluated by calling the evaluate() or evaluate_as() functions. evaluate_as() only differs in that the expected return type is explicitly provided as a template parameter, to allow for different evaluation behavior for different return types. For most use cases, evaluate() will be the right one to use.

Implicit evaluation can apply when BOOST_YAP_CONVERSION_OPERATOR_TEMPLATE is defined. Note that this macro only changes the definition of expression<>, though. Your custom expression templates will not be affected. See the assignment to d1 in Lazy Vector for an example.
